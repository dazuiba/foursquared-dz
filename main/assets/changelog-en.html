What's new in v2010-10-05?

  • Badge description now being shown with badge-unlock in post-checkin dialog.
  • Post-checkin scores were not being displayed. Scores were still being recorded for users on the foursquare servers, just not being shown in the results dialog.
  • Venue shortcut-creation bug fixed.
  • URLs now being linked in tip/to-do details screen.

What's new in v2010-09-30

  • Redesign for most activity screens.
  • Tips and To-do screens added as top-level tabs.
  • Friend information added to venue pages.
  • Can access friend requests from Me tab.
  • Can now refresh Me tab.
  • Switched from xml to json for API calls.

What's new in v2010-08-31

  • Option for install to SD card (Android 2.2 and above required).
  • Added profile settings information to Preferences page. This is a convenience to let users see what email address and name they're using with their Foursquare account.
  • SSL support.

What's new in v2010-08-05

  • Click your user photo in the 'Me' tab to set a new photo for yourself.
  • Added an option in preferences to use a Foursquare image viewer for viewing user photos. Un-check "Image Viewing" to use it instead of your phone's native image viewer (default).

What's new in v2010-07-13

  • Bug fix for new category icons (from 2010-07-12 version) which could cause a force-close on the Places tab.

What's new in v2010-07-12

  • Friend requests screen now has an input filter at top, should make it easier to locate specific friend requests by name.
  • Places list items now have a 'special' icon when the venue is offering a special. Also now showing a person icon when the venue is trending.
  • High res category icons being used now.
  • Added last n system logs messages to feedback emails to help diagnose application errors.

What's new in v2010-06-28

  • Showing recent nearby friend checkins on a map. You can find this on the Friends tab, Menu -> More -> Map.
  • Add Friends screen now has an option to find Facebook friends that are also using Foursquare. NOTE: There's a render bug in Android for the Facebook login page which makes the username and password fields misalign when the virtual keypad is displayed, this should be fixed in Android Froyo.

What's new in v2010-06-16

  • The venue activity has a new menu option called 'Edit Venue' which gives you the option to propose venue edits. These include marking a venue as closed, duplicate, or mislocated. The last option allows you to update all venue information such as address, name, etc.
  • Menu for Friends tab updated to have a 'more' item, which gives the option to sort friend checkins by time or distance.
  • Add friends and find friends items added to 'more' item on Friends tab menu.
  • Refresh menu item moved to first position in Places tab to be consistent with Friends tab.
  • Fixed bug introduced in v2010-06-07 which was formatting the times of checkins incorrectly in the Friends tab.

What's new in v2010-06-07

  • Now showing the 'add friend' option on user details screens, if not already friends. Should make it easier to build your network.
  • Checkin pings support. A service will wake every N minutes (user-configurable) to check for new checkins from friends.
  • New item on user details pages named 'pings', allows you to turn pings on/off per user. Only applicable if you have enabled ping notifications for yourself.
  • Fix a widget bug that would cause "Application Not Responding" errors.
  • Fixed bug for new users on 1.5. The empty friends screen would crash on some android 1.5 devices.

What's new in v2010-05-11

  • Friend invites page now has an option to generate a list of contacts not on foursquare for sending email invites to join. This should make it easier to get friends playing along.
  • For new users, the Friends tab will now show a sad face and two buttons 'add friends' & 'friend requests'. This should make it a little easier for new users to start adding friends.
  • Updated default message for venue pages when there are no recent checkins. This was done to make it clear that the venue page is only showing recent checkins, not a total checkin history.
  • Sorting nearby venues by popularity and distance instead of just distance, read here for more info.
  • Updated to SDK level 6.
